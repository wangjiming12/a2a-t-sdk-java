# 1 a2a-t-sdk-java 用户指南

## 1.1 特性介绍
### 1.1.1 什么是A2A-T？
A2A-T（Agent-to-Agent Telecom）是基于A2A协议推出的电信领域多智能体互联协议，专为电信领域复杂协同场景设计。
业界通用智能体互联协议主要关注智能体互联和交互框架，对业务场景和具体交互内容关注不足，导致任务完成的成功率不高。电信领域的业务场景复杂、要求高，运维智能体互联和协同需要有专门的协议进行支撑。A2A-T方案基于A2A协议，重点针对电信领域业务流相关的信息模型、任务协商、协作安全等增强能力进行应用扩展。

### 1.1.2 A2A-T SDK与A2A SDK的关系
 
A2A-T协议是基于A2A协议的扩展，针对协议扩展的内容提供了A2A-T SDK，支撑快速构建满用于电信领域复杂协同场景的智能体。A2A-T SDK独立于A2A SDK，通过同时集成A2A-T SDK与A2A SDK可以构建支持A2A-T协议的agent,支持电信领域多agent之间的确定性、高可靠、高效及安全协同。

### 1.1.3 能力介绍

a2a-t-sdk-java 是 A2A-T 协议的 Java SDK 实现，提供客户端提示词生成、服务端提示词校验和协商运行时能力。

主要能力包括：

- **客户端提示词生成**：通过 `A2ATClient` 将自然语言或结构化输入转换为A2A-T协议报文。
- **服务端提示词校验**：通过 `A2ATServer` 校验A2A-T协议报文 的场景、槽位和语义一致性。
- **协商流程**：支持 `information`协商。
- **模块化 Maven 工程**：按 core、resources、llm、prompt、negotiation、client、server、sample 分层组织。
- **A2A Java 集成样例**：`a2a-t-sample` 基于 `a2a-java v1.0.0.Beta1` 演示真实 HTTP+JSON/REST 链路。

## 1.2 应用场景

Java SDK 通常用于以下场景：

1. Java 客户端 Agent 接收用户意图，生成 A2A-T 协议报文。
2. Java 服务端 Agent 接收 A2A 请求，校验 prompt 后进入业务执行。
3. 客户端和服务端通过协商上下文进行多轮信息补充或任务可行性确认。
4. 与注册中心配合，实现 AgentCard 注册、查询和 A2A 调用地址解析。

## 1.3 环境要求

| 项目 | 要求 |
| --- | --- |
| JDK | 17+ |
| 构建工具 | Maven 3.8+ |
| LLM | 可访问的 OpenAI-compatible 服务及 API Key |
| 注册中心 | 运行 Java sample 时需要先启动 registry-center |
| A2A Java | sample 基于 `a2a-java v1.0.0.Beta1` |

## 1.4 编译安装

在仓库根目录执行：

```bash
cd {项目路径}/a2a-t-sdk-java
mvn -DskipTests package
```

如需使用本地 Maven 仓库目录：

```bash
mvn "-Dmaven.repo.local=.mvn/repository" -DskipTests package
```

运行测试：

```bash
mvn test
```

## 1.5 准备配置

仓库根目录提供 `client.env` 和 `server.env`，可直接按本地环境修改。

客户端配置示例：

```properties
A2AT_LANGUAGE=zh-CN
A2AT_PROMPT_SOURCE_TYPE=classpath
A2AT_PROMPT_RESOURCE_LOCAL_ROOT_DIR=
A2AT_LLM_PROVIDER=openai
A2AT_LLM_MODEL=deepseek-chat
A2AT_LLM_BASE_URL=https://api.deepseek.com
A2AT_LLM_API_KEY={your_api_key}
A2AT_NEGOTIATION_STATE_STORE_TYPE=in_memory
REGISTRY_CENTER_HOST=127.0.0.1
REGISTRY_CENTER_PORT=5001
```

服务端配置示例：

```properties
A2AT_LANGUAGE=zh-CN
A2AT_PROMPT_SOURCE_TYPE=classpath
A2AT_PROMPT_RESOURCE_LOCAL_ROOT_DIR=
A2AT_LLM_PROVIDER=openai
A2AT_LLM_MODEL=deepseek-chat
A2AT_LLM_BASE_URL=https://api.deepseek.com
A2AT_LLM_API_KEY={your_api_key}
A2AT_NEGOTIATION_STATE_STORE_TYPE=in_memory
A2AT_SAMPLE_HOST=127.0.0.1
A2AT_SAMPLE_PORT=26335
REGISTRY_CENTER_HOST=127.0.0.1
REGISTRY_CENTER_PORT=5001
```

## 1.6 快速运行示例

Java 示例模块位于 `a2a-t-sdk-java/a2a-t-sample`。

### 1.6.1 前置条件

1. 启动注册中心服务。
2. 确认 `server.env` 中的 `REGISTRY_CENTER_HOST` 和 `REGISTRY_CENTER_PORT` 指向注册中心。
3. 确认 `client.env` 中的注册中心地址与服务端一致。
4. 填写 `client.env` 和 `server.env` 中的 `A2AT_LLM_API_KEY`。

### 1.6.2 编译 sample

```bash
cd {项目路径}/a2a-t-sdk-java
mvn "-Dmaven.repo.local=.mvn/repository" -pl a2a-t-sample -am -DskipTests package
```

### 1.6.3 启动服务端

```bash
java @a2a-t-sample/target/server.javaargs.txt
```

服务端启动后会：

- 构造示例 AgentCard。
- 向注册中心注册 AgentCard。
- 启动本地 A2A HTTP 服务。
- 等待客户端请求并校验 A2A-T prompt。

### 1.6.4 启动客户端

另开终端执行：

```bash
java @a2a-t-sample/target/client.javaargs.txt
```

客户端会：

1. 从 `sample/client/scenario.json` 读取示例输入。
2. 根据配置查询注册中心中的目标 AgentCard。
3. 使用 `A2ATClient` 生成 processed task prompt。
4. 构造 A2A HTTP+JSON/REST 请求。
5. 接收服务端任务状态、消息和 artifact 事件。


## 1.7 配置文件速查

| 配置项 | 说明 |
| --- | --- |
| `A2AT_LANGUAGE` | 提示词资源语言，常用 `zh-CN` |
| `A2AT_PROMPT_SOURCE_TYPE` | 提示词资源来源，支持 `classpath` 和 `local_file`，默认 `classpath` |
| `A2AT_PROMPT_RESOURCE_LOCAL_ROOT_DIR` | 本地提示词资源根目录 |
| `A2AT_LLM_PROVIDER` | LLM 提供方，sample 使用 `openai` |
| `A2AT_LLM_MODEL` | 模型名称 |
| `A2AT_LLM_BASE_URL` | LLM 服务地址 |
| `A2AT_LLM_API_KEY` | LLM API Key |
| `A2AT_NEGOTIATION_STATE_STORE_TYPE` | 协商状态存储，当前支持 `in_memory` |
| `A2AT_SAMPLE_HOST` | sample 服务端监听地址 |
| `A2AT_SAMPLE_PORT` | sample 服务端监听端口 |
| `REGISTRY_CENTER_HOST` | 注册中心地址 |
| `REGISTRY_CENTER_PORT` | 注册中心端口 |

## 1.8 约束与限制

1. 当前版本为 Alpha，接口和模块组织可能随版本演进调整。
2. 协商状态当前使用内存存储，不提供跨进程持久化。
3. `A2ATClient` 和 `A2ATServer` 不自动查找 `.env`，调用方必须显式传入配置文件路径。
4. SDK 不提供认证鉴权、密钥管理和 AgentCard 注册中心服务能力。
5. sample 依赖注册中心完成 AgentCard 注册和查询，单独运行客户端前需要先启动服务端并完成注册。
